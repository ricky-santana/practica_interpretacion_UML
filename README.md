# Práctica de Interpretación UML a Java ☕

Busqué un diagrama UML para practicar su lectura e interpretación. En este ejercicio obvié la lógica, quería solo coger soltura interpretando estos gráficos. Este repositorio contiene la implementación en Java de un sistema de pedidos basado en un diagrama de clases.

### Conceptos de POO Aplicados:
* **Herencia:** Implementada en la jerarquía de `Payment` (`Cash`, `Credit`, etc.).
* **Asociaciones Bidireccionales:** Relaciones entre `Order`, `Customer` e `Item`.
* **Enumeraciones:** Uso de `OrderStatus` para el control de estados.
* **Multiplicidad:** Uso de `ArrayList` para representar relaciones `*` y `1..*`.
